# Contributing

Thanks for your interest in contributing! We welcome bug reports, feature requests, documentation improvements, and code changes.

How to contribute

1. Fork the repository.
2. Create a named branch for your change: `git checkout -b feature/short-description`.
3. Make your change with clear, focused commits.
4. Add or update tests where applicable.
5. Open a pull request against the `main` branch describing your change.

Guidelines

- Follow the `CODE_OF_CONDUCT.md`.
- Keep changes small and focused; one logical change per PR.
- Write clear commit messages and PR descriptions.

Code style

- This project is a simple static site (HTML/CSS/JS). Add linting changes to `package.json` if applicable.

Reporting security issues

If you discover a security vulnerability, please see `SECURITY.md` for how to report it privately.
